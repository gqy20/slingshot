# T003 · 碰撞力为什么不是一个数？

## 文档状态

- 状态：候选 Episode，可与 T001/T002 并行做模型原型
- 建议时长：5 分钟
- 主要受众：理工科大学生、研究生、工程师、科学爱好者
- 主要渲染：Godot
- 可复用基础：现有撞击事件、慢动作、冲量与平均力显示
- 新增能力：连续曲线轨道、接触形变模型、高频采样、面积高亮

## 一句话承诺

让同一个冲量变成几条形状完全不同的力—时间曲线，解释为什么“平均力”和“峰值力”不是同一个答案，也不是 Godot 一次碰撞回调就能直接给出的数。

## 观众认知转折

```text
进入前：碰撞力就是撞击发生那一刻的一个很大的数。
第一层：碰撞力会在接触期间上升、达到峰值、再下降。
第二层：动量变化只决定力—时间曲线的面积，不决定曲线形状。
第三层：平均力依赖接触时间，峰值还依赖材料、几何和测量带宽。
离开时：问“碰撞力是多少”之前，必须先问测量的是什么量、在哪段时间内测量。
```

## 开场钩子

前 15 秒并排播放两次慢动作撞击：

1. 同一个球、同一个入射速度；
2. 左侧接触硬挡板，右侧接触软垫；
3. 两侧最终动量变化设置为相同；
4. 左侧力曲线窄而高，右侧宽而低；
5. 冻结后只显示：`面积相同，峰值不同。`

开场不先解释冲量公式，让观众先看到“同一个结果可以隐藏不同过程”。

## 科学问题

### 冲量约束

一维情况下：

```text
J = Δp = m(v_after - v_before)
J = ∫ F(t) dt
```

如果入射与离开速度相同，两次碰撞的冲量相同，因此两条力—时间曲线面积相同。但这不能唯一确定：

- 接触持续时间；
- 峰值力；
- 上升和下降速率；
- 是否存在多次振荡或二次接触。

### 平均力

在明确的接触区间 `Δt` 内：

```text
F_avg = J / Δt
```

它是区间平均值，不是瞬时峰值。改变接触区间的定义也可能改变报告的平均值，因此画面必须同时标出积分区间。

### 构造曲线

第一层解释优先使用数学上可控的力脉冲，而不是立即声称模拟了真实材料。

例如半正弦脉冲：

```text
F(t) = F_peak sin(πt/T),  0 ≤ t ≤ T
J = 2F_peak T / π
F_peak = πJ / (2T)
```

固定冲量 `J` 时，接触时间 `T` 越短，峰值越高。这一关系可以精确地驱动画面和面积填充。

### 示例接触模型

第二层再引入声明清楚的弹簧—阻尼接触模型：

```text
δ = penetration depth
F_contact = max(0, kδ + cδ_dot)
```

或者使用非线性弹性项：

```text
F_contact = max(0, kδ^(3/2) + damping_term)
```

首版只需要选择一种并说明它是示例模型。不能用调出来的 `k`、`c` 冒充某种真实材料的精确参数。

### 损伤与保护

软垫延长接触时间通常能降低给定冲量下的峰值力，但实际损伤还依赖：

- 受力面积与应力分布；
- 材料强度和断裂韧性；
- 形状、约束和局部应力集中；
- 应变率；
- 能量吸收、塑性变形和多轴载荷。

本集不把“峰值力更低”直接等同于“绝对不会损坏”。

## Godot 物理引擎数据的边界

当前项目在碰撞前后读取速度，并通过：

```text
J = Δp
F_avg ≈ |J| / Δt_sample
```

给出平均力估计。这个值适合解释离散采样下的动量变化，但不能直接命名为材料的真实瞬时峰值力。

Godot 刚体求解器为了满足非穿透、反弹和摩擦约束，会在离散物理步中施加冲量。单步冲量除以物理 Tick 并不自动等价于真实接触过程中的力传感器曲线。

本集应把这一差异直接变成内容，而不是隐藏实现限制。

## 视觉设计

### 左右因果绑定

- 左侧或上方展示物体形变和速度；
- 右侧或下方展示同步力—时间曲线；
- 接触形变量增加时，曲线上的光标同步向上移动；
- 反弹结束时，面积填充完成；
- 不允许曲线与碰撞动画只是同时播放却没有逐帧对应。

### 曲线语义

- 横轴必须标时间单位；
- 纵轴必须标力单位；
- 峰值使用一个点和数值；
- 平均力使用同面积矩形表达；
- 冲量使用曲线下面积表达；
- 三个量不得共用一个模糊的“大力”标签。

### 时间放大

真实接触可能远短于一个视频帧。画面应明确出现时间放大倍率或局部时间轴，避免观众误以为软垫形变真的持续数秒。

### 沉浸与测量

- 开场碰撞使用全屏对象和极少文字；
- 进入曲线分析后再切到测量布局；
- 顶部系列栏与固定图例不应占据整个五分钟；
- 结果不是传统排行榜，不设置“赢家”。

## 声音设计

- 硬接触使用短促、高频、衰减快的声音；
- 软接触使用更长、更低频的压缩和恢复声；
- 两次声音只表达触感差异，不声称是真实材料声学合成；
- 曲线光标移动不连续打点，避免把高频采样变成噪声；
- 低频采样漏峰值时，可用声音瞬态与稀疏采样点形成反差。

## 模拟与数据设计

### 两级数据源

第一层使用解析力脉冲，确保“同冲量、不同峰值”是严格可控的演示：

```text
impulse_pulse_record
  time_sec
  force_n
  cumulative_impulse_ns
```

第二层使用独立接触模型生成示例曲线：

```text
contact_model_record
  penetration_m
  relative_velocity_mps
  contact_force_n
  object_momentum_kg_mps
  elastic_energy_j
  dissipated_energy_j
```

不要把 Godot 通用刚体碰撞回调当作第二层的权威连续力曲线。

### 采样率

- 接触求解步长应由刚度和稳定性测试确定，不能沿用视频 30 FPS；
- 模拟可使用远高于现有 120 Hz 的独立固定步长；
- RunRecord 保留高频曲线或经过防混叠的降采样版本；
- 视频回放按绝对时间插值；
- 用不同虚拟传感器采样率生成“漏掉峰值”的对比。

### 结果指标

```text
impulse_ns
contact_duration_sec
average_force_n
peak_force_n
time_to_peak_sec
max_penetration_m
restitution_coefficient
energy_dissipated_j
sampled_peak_force_n
```

## 科学验证

- 解析脉冲数值积分所得面积应与目标冲量一致；
- 动量变化应与累计接触冲量在容差内一致；
- 缩短解析脉冲持续时间时，峰值应按公式变化；
- 接触模型不得在已经分离后产生拉力；
- 时间步减半后，冲量、峰值和接触时间应收敛；
- 不同采样率的模拟必须基于同一条高频参考曲线；
- 平均力的区间起止点必须写入 sidecar。

## 制作验收

- 静音观看开场，能看出两条曲线面积相同而峰值不同；
- 视频始终区分冲量、平均力和峰值力；
- 不把单个 Godot Tick 的冲量直接称为真实峰值力；
- 不把峰值力作为损伤的唯一决定因素；
- 360 像素宽观看时，曲线形状、峰值和面积仍可辨认；
- 结尾观众能复述：`动量变化决定面积，接触过程决定曲线形状。`

## 需要新增的系统能力

- 通用标量曲线轨道与累计积分填充；
- 解析脉冲生成器；
- 独立高频接触求解器；
- 高频数据的确定性降采样与回放；
- 峰值、平均值、积分面积三种视觉标注；
- 局部慢动作倍率和真实时间标尺；
- 模拟量与测量量的命名审计。

## 暂不包含

- 真实材料有限元分析；
- 三维接触斑和应力张量；
- 裂纹萌生与断裂传播；
- 精确声学合成；
- 根据卡通目标外形预测真实损伤；
- 把保护装备设计简化为只优化峰值力。
