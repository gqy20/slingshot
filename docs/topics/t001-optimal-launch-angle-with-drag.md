# T001 · 45° 真的总是飞得最远吗？

## 文档状态

- 状态：候选样片，可进入模型原型与分镜阶段
- 建议时长：5 分钟
- 主要受众：理工科大学生、研究生、工程师、科学可视化爱好者
- 主要渲染：Godot
- 可复用基础：现有角度扫描 Episode、弹弓、轨迹比较、结果排名
- 新增能力：显式空气阻力、数值积分、后台寻优、沉浸镜头、参数曲线

## 一句话承诺

先让 45° 在教科书条件下获胜，再把空气放回世界里，亲眼看见最佳角度移动，并追问它究竟由哪些参数决定。

## 观众认知转折

```text
进入前：45° 是抛体运动的最佳角度。
第一次验证后：在真空、同速、同高落地时，45° 确实获胜。
第二次实验后：加入空气阻力后，最佳角度不再固定为 45°。
离开时：所谓“最佳角度”永远属于一组明确的模型和参数。
```

这集不以“教材错了”为结论。教材给出的结果在声明的理想条件下正确；视频的价值是展示条件改变后如何重新寻找答案。

1. 真空中五个角度同时发射，45° 获胜；
2. 画面出现“把空气放回来”，同一组物体再次发射；
3. 在新赢家即将落地时冻结，不立即显示具体角度；
4. 提问：改变答案的究竟只是“有空气”三个字吗？

开场不出现推导卡、顶部系列栏或常驻图例。角度标签直接附着在发射方向或物体附近。

## 科学问题

### 理想基线

在以下条件同时成立时：

- 重力场均匀；
- 无空气阻力；
- 初速度大小相同；
- 发射点和落地点等高；
- 地面水平；
- 物体可视为质点；

水平射程为：

```text
R = v₀² sin(2θ) / g
```

因此 `θ = 45°` 时达到最大值。

### 空气阻力模型

首版采用常数阻力系数的二次阻力：

```text
v_rel = v - v_air
F_d = -1/2 ρ C_d A |v_rel| v_rel
a = g + F_d / m
```

首版主实验设置 `v_air = 0`，避免把风速和空气阻力混成一个问题。风可以只在结尾作为下一层变量出现。

### 归纳参数

定义弹道系数：

```text
β = m / (C_d A)
```

在其余条件相同时，较大的 `β` 表示阻力相对于惯性更弱。视频后半段不分别扫描所有质量、尺寸和阻力系数，而是先展示它们如何通过 `β` 共同进入问题。

### 不做的物理声明

- 不声称常数 `C_d` 能覆盖全部雷诺数；
- 不模拟自旋、Magnus 力、升力或湍流尾迹；
- 不把二维球形近似当作卡通小鸟的真实空气动力；
- 不在求解前写死“最佳角度一定小于某个具体数值”。

## 初始参数建议

以下参数只用于原型，最终值以轨迹可读性和数值审计结果为准：

```json
{
  "gravity_mps2": 9.81,
  "air_density_kg_m3": 1.225,
  "wind_mps": [0.0, 0.0],
  "projectile_mass_kg": 0.2,
  "projectile_radius_m": 0.05,
  "drag_coefficient": 0.47,
  "launch_speed_mps": 25.0,
  "launch_height_m": 0.0,
  "landing_height_m": 0.0
}
```

选择参数的目标是让真空和有阻力的最佳角度差异肉眼可见，同时保持轨迹仍像普通抛体，而不是被阻力迅速压扁。

## 视觉设计

### 沉浸镜头

- 开场比赛占满逻辑画布；
- 地面只保留一条基准线，不显示实验室面板；
- 发射物至少占手机画面可辨识尺寸；
- 真空轨迹用低透明虚线，有阻力轨迹用实体数据色；
- 阻力箭头长度与 `|v_rel|²` 的映射必须单调并设视觉上限。

### 测量镜头

- 角度—射程曲线成为画面主体，不缩在 HUD 角落；
- 最大点由移动光标和切线变化揭示；
- 公式项应由速度分量和曲线标记迁移形成，而不是整张公式卡淡入；
- 条件清单只在结论阶段出现。

### 颜色语义

- 冷色：低发射角；
- 暖色：高发射角；
- 琥珀强调色：问题、当前最大值和模型条件；
- 真空/参考解通过线型和透明度区分，不另占一套数据色。

## 声音设计

- 第一次发射使用干净、短促且一致的释放声；
- “把空气放回来”时加入极轻的空气底噪，不使用夸张风暴声；
- 阻力箭头增强时可以使用与速度连续映射的窄带风声；
- 扫描曲线的采样点用稀疏、音高上升后下降的提示音；
- 最大值确定时只给一次明确落点声，不连续堆叠胜利音效。

## 模拟与数据设计

### 求解器

- 所有权威状态使用 SI 单位；
- 使用四阶 Runge–Kutta 或精度相当的固定步长积分；
- 原型步长建议 `1/600 s`，再通过收敛测试决定正式值；
- 落地发生在两个采样点之间时，对交点时间和水平位置插值；
- 画面像素只由回放层映射，不进入空气动力计算。

### 最优角度搜索

1. 在合理角度范围内做粗扫描；
2. 找到粗略最大值附近区间；
3. 使用更细扫描或一维寻优细化；
4. 报告搜索分辨率与数值容差；
5. 画面可只播放五条代表轨迹，但结果曲线必须来自完整扫描。

## 科学验证

- 关闭空气阻力后，与解析抛体轨迹和 `45°` 最大值比较；
- 时间步减半后，最佳角度和射程变化必须低于规定容差；
- 阻力功率不得为正；
- 累计耗散能量不得下降；
- 同一参数、版本和步长重复运行应得到同一结果；
- 至少用一个独立参考实现或高精度离线计算复核代表参数。

## 制作验收

- 静音观看前 15 秒，仍能看懂“两次比赛结果不同”；
- 360 像素宽缩略播放时，发射物、轨迹和赢家仍可辨认；
- 正片前 30 秒不出现长公式卡；
- 视频不得说“45° 是错的”，而应说清它成立的条件；
- 最佳角度来自数值结果，不由脚本预设；
- 结尾能用一句话复述：`最佳角度属于模型条件。`
